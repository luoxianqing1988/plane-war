# 数字飞机大战 — 项目计划

## 阶段一：静态战场搭建

**目标**：打开浏览器能看到完整的游戏界面布局

- [ ] 创建项目目录结构（`index.html`、`style.css`、`js/`）
- [ ] 竖屏布局：顶部积分栏 + 基地区 + 战场区域 + 底部留白
- [ ] Canvas 绘制天空背景（浅蓝→深蓝渐变）
- [ ] 基地绘制（占位图形，正常状态）
- [ ] 在右侧 5 列网格位置上静态摆放 3 种敌机（用彩色矩形/圆角形状占位）
- [ ] 积分和血量文字显示

**可验证**：浏览器打开页面，看到完整的战场画面。

---

## 阶段二：敌机出动 + 回合循环

**目标**：游戏能自动运转回合，敌机生成、移动、出兵规则全部跑通

- [ ] 回合状态机实现（出兵→移动→攻击判定→下一回合）
- [ ] 敌机按权重随机生成，上限 4 架，出兵冷却逻辑
- [ ] 敌机每回合向左移动一格（后台格子概念）
- [ ] 敌机到达第 1 格时消失（先不实现扣血，留到阶段四）
- [ ] 多个飞行航线（上下随机）
- [ ] 状态信息以文字日志形式在页面上显示（便于调试）

**可验证**：刷新页面，看到敌机一回合接一回合地生成、向左移动、抵达左侧后消失。

---

## 阶段三：答题系统

**目标**：玩家可以真正"玩游戏"——点击敌机答题

- [ ] 穷举题库生成（加法 ≤5 / 减法 ≤5 / 加减混合 ≤10）
- [ ] 点击敌机进入答题模式（背景变灰、题目浮现）
- [ ] 10 秒倒计时条（绿→黄→红渐变）
- [ ] 2×2 答案按钮网格
- [ ] 答对：敌机消失 + 积分增加 + 激光特效（代码绘制）
- [ ] 答错/超时：反馈提示，退出答题
- [ ] 每回合只能答一次题，退出后进入攻击判定阶段

**可验证**：能完整玩一回合——点敌机、答题、看到对错反馈。

---

## 阶段四：完整游戏闭环 + 特效 + 素材集成

**目标**：游戏完整可玩，有头有尾

- [ ] 敌机抵达第 1 格时攻击基地（黄色光球动画、扣血）
- [ ] 基地血量变化：受损切换、闪红效果
- [ ] 基地血量归零时播放爆炸动画，游戏结束
- [ ] 爆炸粒子特效（代码绘制）
- [ ] 激光从基地射向敌机（代码绘制）
- [ ] 结算界面：本轮积分、历史最高分（localStorage）、再来一局
- [ ] 集成你提供的图片素材和音效文件

**可验证**：完整游戏流程——出兵→答题→敌机攻击→基地扣血→游戏结束→结算→再来一局。

---

## 约定说明

- 每个阶段完成后经确认再进入下一阶段
- 美术素材和音效文件由甲方提供，放入 `assets/` 目录
- 激光、光球、爆炸特效由代码（Canvas）绘制
- 先以占位图形开发，素材到位后一键替换
